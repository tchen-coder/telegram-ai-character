# 色情RP关系系统 - PRD设计文档（V2.0 - 2025/2026 更新版）

**文档目的**  
为色情向角色扮演对话系统设计一套稳定、可配置、商业化友好的关系推进机制。  
核心原则：阶段只升不降、负反馈极弱、支持付费加速、计算频率可配置。

**版本历史**  
- V1.0：基础三阶段 + 底线保护 + 单轮实时计算  
- V2.0：增加计算频率配置 + 预留付费加速字段（PaidBoostRV）

## 1. 核心约束（不可妥协）

| 约束项                  | 说明                                                                 |
|-------------------------|----------------------------------------------------------------------|
| 关系阶段数量            | 固定3个：朋友 → 恋人 → 爱人                                          |
| 阶段是否可逆            | **完全不可逆**：一旦解锁更高阶段，永久持有该阶段及以上权限           |
| 单次最大负向变化        | **-3**（最强拒绝也只扣3分）                                          |
| 阶段底线保护            | 恋人阶段最低40，爱人阶段最低70，即使扣分也强制回保底值               |
| 阶段阈值                | 朋友 → 恋人：≥40<br>恋人 → 爱人：≥70                                 |
| 初始值                  | 新对话 RV = 15（朋友阶段）                                           |
| 计算触发时机            | **可配置**：每 N 轮用户消息触发一次关系更新（默认 N=1，即实时）      |
| 付费加速预留字段        | PaidBoostRV（暂不启用逻辑，仅保留字段与展示）                        |

```json5

    "max_unlocked_stage": 0,

    // 阶段名称（展示用，可多语言）
    "stage_names": ["朋友", "恋人", "爱人"],

    // 每个阶段的RV保底值
    "stage_floor_rv": [0, 40, 70],

    // 升级阈值（可未来做活动调整）
    "stage_thresholds": [40, 70, 100],

    // 上一次更新的RV值（用于对比变化）
    "last_rv": 15,

    // 上一次更新的delta
    "last_delta": 0,

    // 上一次更新发生在第几轮
    "last_update_at_turn": 0,

    // 付费后可选择：1. 直接加到 current_rv  2. 替换 current_rv  3. 作为保底值
    "paid_boost_rv": 0,              // 当前为0，未付费
    "paid_boost_applied": false,     // 是否已应用过付费值
    "paid_boost_source": null,       // "gift"、"subscription"、"one_time_purchase" 等

    // 可选：情感摘要（防上下文丢失）
    "emotion_summary": {
      "last_updated_turn": 0,
      "text": "",
      "adjustment_factor": 0.0       // 未来可用于微调，如长期高甜+1.5
    }
  }

4. 关系更新流程（带频率控制）

每收到一条用户消息 → turn += 1

IF (turn % update_frequency == 0) OR (turn == 1) THEN
    触发关系计算：
    1. 取最近 M 轮（建议8~20轮）用户消息 + 本轮消息
    2. LLM 打分 → 本轮 delta ∈ [-3, +15]
    3. 如果有 pending_delta_accumulator，累加：total_delta = pending + 当前delta
    4. 清空 pending_delta_accumulator
    5. tentative_rv = current_rv + total_delta
    6. new_rv = max(当前阶段floor, min(100, tentative_rv))
    7. 判断是否升级阶段（只升不降）
       - 如果 new_rv >= 70 且 max_unlocked_stage < 2 → 升至2
       - 如果 new_rv >= 40 且 max_unlocked_stage < 1 → 升至1
    8. 更新 current_rv = new_rv
    9. 更新 max_unlocked_stage / current_stage / floor
    10. 记录 last_delta / last_rv / last_update_at_turn
ELSE
    把本轮 delta 累加到 pending_delta_accumulator
    （暂不更新RV和阶段，等待下次批量计算）
END

频率配置建议值

update_frequency适用场景优点缺点1标准色情RP、追求快速反馈实时、爽感强波动较大2～3中等节奏、希望稍微平滑波动减小升级稍慢4～6长线养成、剧情向、重度用户变化更稳定、更有仪式感付费转化窗口变长8+极慢热、土味剧情、测试用几乎无波动用户可能觉得没进展
5. 付费加速字段（PaidBoostRV）说明
目前仅做字段保留 + 前端展示准备，暂不接入实际付费逻辑。
设计意图：

用户通过付费（礼物、会员、单次解锁）获得一个“可兑现的更高亲密度”
展示时可提示：“当前亲密度15，您的专属加速值已达68（恋人阶段）”
未来可支持三种兑现方式（待产品决定）：
直接加成：current_rv += paid_boost_rv（或部分）
覆盖：current_rv = max(current_rv, paid_boost_rv)
保底：把 paid_boost_rv 作为新的 floor（更保值）


字段更新时机（预留）：

付费成功回调时写入 paid_boost_rv
首次应用后设置 paid_boost_applied = true
可记录来源，便于后续运营分析
